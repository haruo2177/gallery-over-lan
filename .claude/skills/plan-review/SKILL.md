# 実装計画レビュー

以下の実装計画をレビューしてください。

**レビュー対象:**
$ARGUMENTS

## 手順

### 1. 計画で言及されたファイルの確認
計画に記載されている変更対象ファイルを実際に読み、計画の前提が正しいか検証してください。

### 2. レビュー観点

以下の観点から計画を評価してください:

**正確性:**
- 参照しているファイル・関数・行番号は実際に存在するか
- 既存コードの動作理解は正しいか
- 変更の影響範囲は正しく把握されているか

**完全性:**
- 変更が必要なファイルに漏れはないか
- エッジケースの考慮は十分か
- 既存テストへの影響は考慮されているか

**設計品質:**
- プロジェクトの設計パターン（MVVM + Clean Architecture）に沿っているか
- 不要な複雑さを持ち込んでいないか
- 既存のユーティリティや関数を適切に再利用しているか

**リスク:**
- 既存機能へのリグレッションの可能性
- パフォーマンスへの影響
- スレッドセーフティの問題

### 3. 出力フォーマット

```
## レビュー結果: ✅ 承認 / ⚠️ 条件付き承認 / ❌ 要修正

## 問題点
- 【重大】（あれば）
- 【軽微】（あれば）

## 改善提案
- （あれば具体的に記述）

## 良い点
- （計画の優れた点があれば記述）
```
