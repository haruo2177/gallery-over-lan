# 実装計画

以下のタスクの実装計画を作成してください。

**タスク:**
$ARGUMENTS

## 手順

### 1. コードベース調査
Exploreエージェントを使って、実装に必要な以下の情報を収集してください:

- 変更対象のファイルと現在の実装内容
- 再利用可能な既存の関数・ユーティリティ・パターン
- 類似機能の実装例（参考にできるもの）

### 2. 実装計画の設計
Planエージェントを使って、収集した情報をもとに実装計画を作成してください。
複雑なタスクの場合、異なる観点（シンプルさ vs パフォーマンス等）で最大3つのPlanエージェントを並列起動してください。

### 3. 出力フォーマット

以下の形式で計画を記述してください:

```
## 背景
- なぜこの変更が必要か

## 変更概要
- 変更ファイルごとに何をするかの一覧

## 詳細な実装手順
1. （ステップ1: ファイルパス、変更内容、変更行の目安）
2. （ステップ2: ...）
...

## 再利用する既存コード
- （ファイルパス:関数名 — 用途）

## リスク・注意点
- （あれば記述）

## 検証方法
- （ビルド、テスト、手動確認の手順）
```
