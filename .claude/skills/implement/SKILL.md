# 実装

以下のタスクを実装してください。

**タスク:**
$ARGUMENTS

## 手順

### 1. 現状把握
まず変更対象のファイルを読み、最新の状態を確認してください。
会話内に実装計画がある場合はそれに従ってください。ない場合はExploreエージェントで調査してから実装してください。

### 2. 実装ルール

- **既存パターンに従う**: プロジェクトの既存コードスタイル・アーキテクチャに合わせる
- **最小限の変更**: タスクに必要な変更のみ行う。関係ないコードの整理やリファクタリングはしない
- **新規ファイルは最小限に**: 既存ファイルの編集を優先する
- **コメント・ドキュメントは変更箇所のみ**: 変更していないコードにコメントを追加しない
- **不要な抽象化をしない**: 1回しか使わないヘルパー関数を作らない

### 3. 実装後の確認

実装が完了したら、変更したファイルを再度読み返して以下を確認してください:
- コンパイルエラーになりそうな箇所がないか
- import文の不足がないか
- 既存コードとの整合性が取れているか

### 4. 出力フォーマット

```
## 変更ファイル一覧
- （ファイルパス: 変更内容の要約）

## 実装のポイント
- （判断が必要だった箇所とその理由）

## 次のステップ
- /review でコードレビューと動作確認を実行してください
```
