# タスク理解と既存実装との矛盾確認

以下のタスクについて、理解と矛盾確認を行ってください。

**タスク:**
$ARGUMENTS

## 手順

### 1. タスク理解
Exploreエージェントを使って、タスクに関連するコードベースの領域を調査してください。
必要に応じて最大3つのExploreエージェントを並列起動し、以下を把握してください:

- タスクが影響する画面・コンポーネント・ファイル
- 関連する既存の関数・ユーティリティ・パターン
- データの流れ（ViewModel → UseCase → Repository など）

### 2. 既存実装との矛盾確認
調査結果をもとに、以下の観点で矛盾や問題がないか確認してください:

- **機能的矛盾**: タスクの要件が既存機能と衝突しないか
- **設計パターンの一貫性**: プロジェクトの既存アーキテクチャ（MVVM + Clean Architecture + Hilt）に沿っているか
- **状態管理**: 既存のStateFlow/UiStateパターンと整合するか
- **命名規則**: 既存のコードベースの命名規則と一致するか

### 3. 出力フォーマット

以下の形式でレポートしてください:

```
## タスク理解
- 目的: （何を達成するか）
- スコープ: （影響範囲のファイルとコンポーネント）

## 関連する既存実装
- ファイル一覧と各ファイルの役割

## 矛盾・懸念事項
- （あれば具体的に記述、なければ「なし」）

## 確認事項
- （ユーザーに確認が必要な曖昧な点があれば記述）
```
